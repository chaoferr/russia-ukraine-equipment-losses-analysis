# 数理统计分析报告

## 1. 引言与研究背景
在冲突数据的记录中，交战国官方机构与第三方开源情报组（如 Oryx, WarSpotting）往往展现出极大的数量级差异。为客观评估真实的战损情况并度量两种统计口径的系统性偏误（Selection Bias），本研究对从2022年起积累的官方战报数据（$X \sim \text{Poi}(\lambda_1)$）与WarSpotting图像验证数据（$Y \sim \text{Poi}(\lambda_2)$）进行了全面的统计对齐与数理测算。

## 2. 核心统计方法链
本研究严格奉行以下统计链路以保障论文级别的严谨度：
1. **点估计**：对装备日损失量计算最大似然估计（MLE），求得基础损失率 $\hat{\lambda}$。
2. **基准置信区间**：利用标准泊松分布假定，给出了 Wald、Score 以及 Exact 条件假定下的 $95\%$ 置信区间。
3. **消除时序偏误的 Bootstrap**：考虑到真实战争中某日的大规模接触会导致随后的时序自相关以及异方差等破坏基本分布独立性的因素，本次使用了重抽样区块长为7的 **Moving Block Bootstrap (BCa)** 来提供抗偏校正下更为稳健真实的置信区间。
4. **统计显著性检验**：采用两独立泊松假定的完全条件精确检验算法（Two-independent Poisson conditional test），计算 H0 分布下差异显著性的双侧 $P\text{-value}$。

## 3. 分析结果与可视化呈现

### 3.1 总体装备损失对比 (Total Conflict Evaluation)
利用极大似然及 Bootstrap BCa 计算，主要核心装甲与空中力量在整体周期差异度显现出两支数据的鸿沟。图 `fig1_mle_comparison_log.png` （使用对数纵坐标）揭示了不同装备类别上的估计下限与上限偏移。
- **坦克 (Tank)**：官方战报报称损失率高达 ~7.68 两/天，而 WarSpotting 经过严格相片验证的数字约为 2.58 两/天。两者处于三倍的标定量级。（详见 `data/stratified_estimation_results.csv` ）。
- **无人机 (Drone)**：表现出了最大的分歧——官方给出了约 205 架/天的恐怖战果；而开源平台因多数微型穿越机损失无法找到有效照片记录或由于其自身残骸微小，验证量仅为 0.52。该“观测偏误”展现到极致。
- 经过检验，全序列内所有主要重型装备 $P < 0.05$（甚至是 $P \approx 0$），强烈拒绝 $H_0$（$\lambda_1=\lambda_2$），即双方数据有着根本性的偏差机制。

### 3.2 战役阶段分层探测 (Stratified Evolution)
为了控制异方差影响，我们将战场粗略划分为“2022早期运动战”、“2023拉锯战”、“2024-2026僵持阵地战”三个纪元。详见 `fig2_stratified_tank.png` 柱状图：
- 在 2022 初期的高烈度环境中，官方报称的坦克损失率为 9.74两/天，验证损失为 4.44两/天。
- 随后到 2024 以后的僵持带中，双方通报比例依然保持固定形态（约 6.79 vs 2.0），但可以观察到相对规模都在逐渐趋于保守化衰退。
- **值得注意的是直升机 (Helicopter)**，在 2023年和 2024年后续时段，其差异的 $P\text{-value}$ 并没有跨过 $0.05$ 的门限（接受 $H_0$），说明后期当双方基本杜绝对方纵深的穿透之后，一旦有直升机坠毁，基本上一定伴随坠机照片的流出（观测概率 $p \to 1$ ），使得官方统计与民间验证在这部分的数据达成了一定程度上的弥合。

### 3.3 时序平滑动量学验证 (14-day MA dynamics)
对日时间切片进行了平滑的观测（见 `fig3_timeseries_tank.png` 和 `fig3_timeseries_APC.png`）。在图表中显示无论是俄发起的顿巴斯春季攻势或者是阿夫迪伊夫卡包围战期间，两条曲线（暗红色的 Official 与 午夜蓝的 WarSpotting）在趋势导数上极为吻合，波峰几乎在同时到来。
这说明官方战报虽然在绝对基数上有扩大常数，但能够反映出真实的“战场强度脉冲”；相反，开源数据受到照片拍摄延误的影响而显得更毛刺，但其基线更沉稳。

## 4. 结论
研究表明，官方时序战报有着较严重的数字偏误（普遍达到图像验证数量的2倍至4倍，在无人机等小型目标上更达百倍之上）。然而其时序上的相对波动以及动态脉冲，依然包含重要且真实的战场信息量。本研究经过重抽样校正直置的各种指标（储存于 data 目录）可以放心地传递到下一个预测（T6）模型工作流阶段作为可信的输入因数。
